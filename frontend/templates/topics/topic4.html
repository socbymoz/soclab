Module 4: Cyber Defence Frameworks

Frameworks help SOC analysts understand attacker behavior, communicate findings effectively, and improve detection coverage. This module covers five essential frameworks.

Sub-Domains / Rooms

🔹 Pyramid of Pain

Created by David Bianco, this model shows how hard it is for attackers to change different types of indicators. The higher you can detect, the more pain you cause the attacker.

Pyramid Levels (bottom to top):
1. Hash Values (Trivial to change): Blocking a specific file hash. Attacker recompiles — new hash. Pain: None.
2. IP Addresses (Easy): Blocking a C2 IP. Attacker uses a different IP or CDN. Pain: Minimal.
3. Domain Names (Simple): Blocking evil.com. Attacker uses Domain Generation Algorithm (DGA) — thousands of domains. Pain: Moderate.
4. Network/Host Artifacts (Annoying): Detecting specific HTTP headers, TLS fingerprints, or registry keys. Harder to change. Pain: Significant.
5. Tools (Challenging): Detecting specific tools like Mimikatz, Cobalt Strike, or Metasploit. Attacker must develop custom tools. Pain: High.
6. TTPs (Tough): Detecting the behavior itself — "adversary uses spear-phishing + PowerShell + living-off-the-land". The attacker must completely change their methodology. Pain: Maximum.

Example: Instead of just blocking hash ABC123 (trivial to bypass), write a YARA rule that detects the specific PowerShell download cradle technique the attacker uses (TTP level). This forces the attacker to completely rewrite their payload delivery.

🔹 Cyber Kill Chain

Developed by Lockheed Martin, the Cyber Kill Chain describes 7 phases of a cyber attack. Understanding each phase helps you detect and stop attacks at different stages.

7 Phases with Real Example (SolarWinds Breach 2020):
1. Reconnaissance: Attacker researches SolarWinds employees, finds Orion build infrastructure, identifies supply chain opportunity.
2. Weaponization: Attacker creates malicious update code that looks legitimate, signed with valid digital certificates.
3. Delivery: Malicious update pushed via SolarWinds' own update mechanism to 18,000 customers.
4. Exploitation: The Orion update runs with legitimate trust — no exploit needed, the victim runs it willingly.
5. Installation: Malware (SUNBURST) installs itself, stays dormant for 14 days, then establishes C2.
6. Command & Control (C2): SUNBURST uses HTTP backdoors to communicate with attacker-controlled domains, disguising traffic as legitimate Orion traffic.
7. Actions on Objectives: Attackers identify high-value targets (US government agencies, Fortune 500 companies), steal email and documents from compromised systems.

Detection opportunity: At phase 6 (C2), anomalous DNS queries to rarely-used domains could have been detected by network monitoring.

🔹 Unified Kill Chain

The UKC combines multiple frameworks into 18 phases covering the complete attack lifecycle from initial reconnaissance to post-exploitation actions. It's more comprehensive than the original Cyber Kill Chain.

UKC Phases (simplified):
Phase A (In): Reconnaissance, Weaponization, Delivery, Social Engineering, Exploitation, Persistence, Defense Evasion, C2
Phase B (Through): Pivoting, Discovery, Privilege Escalation, Execution, Credential Access, Lateral Movement
Phase C (Out): Collection, Exfiltration, Impact, Objectives

Example: Ransomware attack mapped to UKC:
Phase A: Spear-phish (Delivery) → macro (Exploitation) → Cobalt Strike (C2)
Phase B: Dump LSASS (Credential Access) → Move to DC (Lateral Movement) → Domain Admin (Privilege Escalation)
Phase C: Deploy ransomware (Impact) → Encrypt all servers (Objectives) → $5M ransom demand

🔹 MITRE ATT&CK

MITRE ATT&CK is a globally accessible knowledge base of adversary tactics and techniques based on real-world observations. It's the industry standard for describing attacker behavior.

MITRE ATT&CK Structure:
- Tactics (14 columns): The "why" — the goal of the technique. Examples: Initial Access, Execution, Persistence, Defense Evasion, Credential Access, Discovery, Lateral Movement, Collection, Exfiltration, Impact
- Techniques (hundreds): The "how" — specific methods to achieve the tactic. Examples: T1566 (Phishing), T1059 (Command & Scripting), T1003 (OS Credential Dumping)
- Procedures: The specific implementation. "APT29 uses spear-phishing with malicious attachments for T1566"

Example Mapping — Ransomware Attack:
| Tactic | Technique ID | Technique Name | Detection | |--------|-------------|----------------|-----------| | Initial Access | T1566.001 | Spear-phishing Attachment | Email gateway filter | | Execution | T1204.002 | User Execution: Malicious File | EDR process monitoring | | Persistence | T1547.001 | Registry Run Keys / Startup | Sysmon Event 13 | | Defense Evasion | T1055.001 | Process Injection | Sysmon Event 8 | | Credential Access | T1003.001 | LSASS Memory | Sysmon Event 10 | | Lateral Movement | T1021.006 | Windows Remote Management | Windows Event 5156 | | Impact | T1486 | Data Encrypted for Impact | File system audit |

🔹 Summit & Eviction

These rooms apply all the frameworks above in hands-on scenarios.

Summit: Analyze a complex real-world attack scenario. You're given PCAPs, logs, and intel reports. You must map the attack to the Cyber Kill Chain, identify the TTPs using MITRE ATT&CK, determine which Pyramid of Pain indicators to block, and recommend defense improvements.

Eviction: You're the incident responder. A threat actor is actively inside your network. Using the frameworks, you must identify their current phase, determine what they've done, and take steps to evict them. You'll block IOCs, identify persistence mechanisms, and implement detection rules to prevent their return.
