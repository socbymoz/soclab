Module 4: Log Analysis & SIEM Operations

Section Overview

Log analysis is the core function of a SOC analyst. This module covers SIEM architecture, log ingestion, search and correlation, alert triage, and threat hunting.

Skills: SPL/KQL/ESQL, log integration, alert triage, threat intel, SOAR. Prerequisites: Modules 1-3. Objectives: Design SIEM, write queries, triage alerts, hunt threats.

Detailed Notes

▸ SubTopic 1: SIEM Architecture & Log Ingestion

Beginner: SIEM collects, normalizes, analyzes logs. Components: Forwarder, Indexer, Search Head. Splunk, Elastic, Sentinel, QRadar, Wazuh. Log flow: Source to Forwarder to Indexer to Analyst.

Intermediate: Syslog (UDP 514), WEF (WinRM), HTTP/S API, Agents (UF/Beats/Wazuh), NetFlow. Each requires specific parsing config.

Advanced: 1-10 TB/day. Hot 30d SSD, Warm 90d HDD, Cold 1yr S3. CIM normalization. Index segmentation. HA clustering. PCI-DSS Req 10, SOC2, HIPAA.

▸ SubTopic 2: Search & Correlation

Beginner: SPL: index=windows EventCode=4625 | stats count by Account_Name | where count > 10. KQL: event.code:"4625". AND, OR, NOT, *, =, !=.

Intermediate: Subsearch, Transaction, Lookups, Time-series, Correlation Rules. Example: RDP brute force detection.

Advanced: ML baselining (anomalydetection), Clustering, UEBA, Risk-based Alerting, Cross-correlation.

▸ SubTopic 3: Alert Triage & IR

Beginner: Acknowledge, check FP, gather context, severity, contain, escalate, document. SLA: Critical=15min, High=1hr, Med=4hrs, Low=24hrs.

Intermediate: Pyramid of Pain: Hash < IP < Domain < Artifacts < Tools < TTPs. CVSS scoring.

Advanced: SOAR automation. Enrich with VT/OTX/MISP. Scope patient zero, lateral movement. Post-incident: update rules, track MTTD/MTTR.

▸ SubTopic 4: Threat Hunting

Beginner: Proactive: Form hypothesis, search, analyze, act. Example: scheduled task persistence hunting.

Intermediate: ATT&CK-based, Intel-based (CISA), Data-driven (outliers), Hypothesis testing.

Advanced: Stack Counting, Baselining, HMM Level 3+, TTP-based hunting, Atomic Red Team validation.

▸ Common Attacks in SIEM

  • Phishing: Email + process creation + DNS correlation.
  • Ransomware: Mass file creates + C2 + process tree.
  • Data Exfiltration: Large HTTP POST + DLP + UEBA.
  • Account Takeover: Impossible travel + unusual device + spray.
  • Web Attack: WAF signatures + anomalous URLs + child processes.
INFO BOX
Master SPL: stats, transaction, eval, rex, lookup, subsearch, timechart, where, table, sort, dedup, eventstats. Covers 90% of SOC queries.
REMEMBER BOX
GIGO: SIEM only as good as its logs. Map every ATT&CK technique to a log source. Verify ingestion before building rules.
WARNING BOX
FPs cause burnout. Threshold alerts, deduplicate, track FP ratio (<5%). Use risk-based alerting for low-confidence events.
SOC ANALYST TIP
Build a SOC Toolkit dashboard: Top talkers, failed login heatmap, process anomalies, new domains, exfiltration candidates. Reduces MTTD by 50%.
INTERVIEW Q&A
Q: "Malicious PowerShell alert — walk through investigation." A: Check Event 4688 command line, decode base64, check parent process (phishing?), process tree, network connections, DNS, VT hash. Block C2, isolate host, escalate.

Intermediate Lab: Sigma Rule + Atomic Red Team Validation

Objectives: Write Sigma rule for WMI lateral movement, convert to Splunk, deploy as saved search, validate with Atomic Red Team.

Steps: Write Sigma rule (WmiPrvSE.exe → cmd.exe) → Convert with sigmac → Deploy in Splunk → Test with Invoke-AtomicTest T1047 → Verify alert fires.

Expected Results: Atomic test generates WMI process creation. Your Sigma detection fires. Extend for LOLBins (mshta, certutil, rundll32).

Advanced Lab: ML Threat Hunting with Jupyter + Elastic

Objectives: Build Python hunting pipeline: query Elasticsearch, detect C2 beaconing via FFT + statistical analysis, visualize in Kibana.

Steps: Query 7 days of network logs via elasticsearch-py → Compute inter-arrival times per flow → Apply FFT for periodicity → Score flows (CV < 0.3, period 10-3600s = C2 candidate) → Push results to Elastic → Kibana scatter plot dashboard.

Expected Results: Identify beaconing flows with regular intervals. Test with malware-traffic-analysis.net PCAPs. Productionize as daily Jupyter notebook with Watcher alerts.

SOC Scenarios

Scenario 1: Multi-Stage Ransomware Detection

What Happened: Alerts: Sysmon Event 1 (WINWORD→cmd→powershell encoded), Event 3 (powershell→45.33.32.156:443), Event 11 (500+ .encrypted files).

Investigate: Check process tree → Decode PowerShell → Check C2 DNS → Check SMB lateral movement → Check email gateway for phishing.

Respond: Isolate host via EDR, block C2 IP, check file servers, disable user, scan email gateway. Document: LockBit ransomware via phishing. MITRE: T1566.001, T1204.002, T1486. Severity: CRITICAL.

Scenario 2: Impossible Travel Account Takeover

What Happened: Sentinel UEBA: User jsmith logged in from NYC (RDP) and Lagos, Nigeria (Azure AD) within 25 min. 5300 miles.

Investigate: Check Azure AD sign-in logs for Nigeria session → MFA method/status → Resources accessed (SharePoint Finance?) → Downloads.

Respond: Disable account, revoke sessions, force password reset, reset MFA, block high-risk countries in CA policy. Document: MFA fatigue attack. MITRE: T1078.004, T1537. Severity: CRITICAL.

Scenario 3: Distributed Port Scan Fire Drill

What Happened: SIEM spikes: 200 alerts/min "ET SCAN SYN Scan" from 50 unique external IPs to web farm.

Investigate: Check Shodan for source IPs (cloud VPS?) → Check if exploitation followed → Check WAF blocks.

Respond: Do NOT block 200 IPs individually. Block ASN if malicious. Add Suricata thresholding. Create SIEM aggregation: >50 IPs on same dest → "Distributed Scan" single alert. Document: Internet background noise. Severity: LOW.
