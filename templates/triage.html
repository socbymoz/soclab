Step 3: Triage

Separating real threats from false alarms
What is Triage?

Triage is quickly evaluating alerts to decide: is this a True Positive (real threat), False Positive (false alarm), or Benign (expected behavior)? Borrowed from emergency medicine — treat the most critical first.

TP = Real attack FP = False alarm BP = Expected behavior
Triage Example
TRIAGE QUEUE — 2026-06-15
═══════════════════════════════════════
[CRITICAL] C2 Beacon       → TP (verified C2 IP in threat intel)
[HIGH]    Log Cleared       → TP (backup_admin, not legitimate)
[HIGH]    New Admin Created → TP (part of known attack chain)
[WARNING] 12 Failed Logins  → TP (brute force, single source IP)
[INFO]    User backup_admin → TP (created by attacker, not IT)
═══════════════════════════════════════
Result: 5/5 TP → Escalate to Investigation
Key question: Does this alert represent actual malicious activity? If yes → investigate. If no → close as FP/BP.
