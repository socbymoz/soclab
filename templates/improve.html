Post-incident analysis and recommendations to prevent the next attack
Why the "Improve" Phase Is the Most Important

Every security incident is a learning opportunity. The Improve phase (also called Post-Incident Activity or Lessons Learned) is where you ensure the same attack cannot happen again.

"A company that experiences a breach and does not learn from it will experience the same breach again — often worse."
The Improvement Cycle
Identify Gaps
What security controls failed?
Prioritize
What should we fix first?
Implement
Deploy fixes and new controls
Validate
Test that fixes actually work
Gaps Found in This Incident

Our investigation revealed several critical security gaps. Each one allowed the attack to progress:

GapWhat AllowedRecommendation
No account lockout policy Attacker brute-forced SSH password (12 attempts in 10 minutes) Lock account after 5 failed attempts for 30 minutes
File upload vulnerability Webshell uploaded to web server Validate file types, restrict upload directory permissions
No MFA on admin accounts Attacker used compromised password to create new admin Require MFA for all administrative access
Unrestricted outbound traffic C2 beaconing and data exfiltration succeeded Deploy firewall with allow-list outbound rules only
No EDR on endpoints Malware execution undetected for hours Deploy EDR on all workstations and servers
No log monitoring alerts Brute force not detected until log review Create real-time alert rules for brute force patterns
Key Insight: The Improve phase closes the loop. Every previous phase (Logs → Alert → Triage → Investigation → Response → Recovery → Report) contributed information that we now use to make the organization stronger. This is called continuous improvement.
Critical Gaps Found
  • No account lockout policy — brute force succeeded
  • File upload vulnerability unpatched
  • No MFA on privileged accounts
  • Outbound traffic not restricted (C2 allowed)
  • No EDR/AV on workstations
Recommendations
  • Implement account lockout after 5 failed attempts
  • Patch file upload validation immediately
  • Require MFA for all administrative access
  • Deploy next-gen firewall with outbound filtering
  • Install EDR on all endpoints (CrowdStrike, SentinelOne)
Detection Improvements
  • NEW Alert on >5 failed SSH logins/min from same IP
  • NEW Monitor new admin account creation (Event ID 4720)
  • NEW Alert on powershell.exe making network connections
  • NEW Detect registry Run key modifications (Sysmon ID 15)
  • NEW Monitor audit log clearing events (Event ID 1102)
