Elaborated Notes with Examples
1. Tactics vs Techniques

MITRE ATT&CK is a knowledge base of adversary behavior. Tactics (the "why") represent the goal — like TA0001 (Initial Access) or TA0005 (Defense Evasion). Techniques (the "how") are specific methods — like T1566 (Phishing) or T1055 (Process Injection). Sub-techniques provide finer granularity.

Example: TA0001 (Initial Access) includes T1566.001 (Spearphishing Attachment). The analyst sees an email with a malicious .docm file. Mapping this to MITRE tells you the adversary's tactic is Initial Access, the technique is Phishing, and the sub-technique is Spearphishing Attachment — informing containment and EDR hunting queries.
2. ATT&CK Matrix Overview

The matrix organizes 14 tactics across columns with techniques listed vertically. Enterprise covers Windows, macOS, Linux, cloud, network, and ICS. Each technique has an ID, name, description, detection recommendations, and mitigation mappings.

Example: An alert for reg.exe add HKLM\SYSTEM\CurrentControlSet\Services\... maps to T1543.003 (Create or Modify System Process: Windows Service). The matrix shows it belongs to TA0003 (Persistence) and TA0004 (Privilege Escalation). The detection section suggests monitoring service creation events (Event ID 4697 / 7045).
3. Key Techniques: T1078, T1059, T1047

T1078 (Valid Accounts) covers using legitimate credentials to bypass auth controls. T1059 (Command and Scripting Interpreter) includes PowerShell, Bash, VBScript. T1047 (Windows Management Instrumentation) covers using WMI for execution, lateral movement, and information discovery.

Example (T1078): A compromised service account svc_backup is used to RDP into a SQL server — Event ID 4624 with LogonType 10. The account has never logged into this server before, indicating credential misuse.
Example (T1059): powershell.exe -Exec Bypass -C "Invoke-Command -Computer DC01 {Get-Process}" — Event ID 4104 captures PowerShell script block logging showing lateral movement.
Example (T1047): wmic /node:DC01 process call create "cmd.exe /c whoami" — Sysmon Event ID 1 shows wmic.exe spawning cmd.exe on a remote system.
4. Mapping Alerts to MITRE

Each SOC alert should be tagged with the relevant MITRE technique(s). This enables trend analysis (most common techniques), coverage gaps, and improved detection rules. Many SIEM integrations (Splunk ES, Azure Sentinel) map rules to MITRE IDs automatically.

Example: An alert triggers: schtasks.exe /create /tn "Updater" /tr "C:\Windows\System32\cmd.exe /c start powershell.exe". The analyst maps this to T1053.005 (Scheduled Task). Adding this to the alert in the SIEM allows the SOC to run a report showing that 40% of incidents this quarter used Scheduled Tasks for persistence, triggering a recommendation to audit all scheduled task creation via GPO.
