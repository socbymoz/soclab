Elaborated Notes with Examples
1. Active Directory — Kerberos / LDAP / GPO

Active Directory is Microsoft's identity management service. Kerberos is the default authentication protocol using tickets (TGT, TGS). LDAP (Lightweight Directory Access Protocol) is used for directory queries and modifications. Group Policy Objects (GPOs) enforce security configurations across domain-joined computers (password policies, software restrictions, audit policies).

Example (Kerberos Attack): Kerberoasting: An attacker requests a TGS ticket for a service account (e.g., MSSQLSvc/SQL01.corp.com) via Add-Type -AssemblyName System.IdentityModel; New-Object System.IdentityModel.Tokens.KerberosRequestorSecurityToken -ArgumentList "MSSQLSvc/SQL01.corp.com". The TGS ticket is encrypted with the service account's NTLM hash. The attacker downloads the ticket with Invoke-Mimikatz -Command '"kerberos::list /export"' and cracks it offline with hashcat: hashcat -m 13100 hash.txt wordlist.txt. Detection: Event ID 4769 with service name + TicketEncryptionType=0x17 (RC4) for unusual accounts.
Example (GPO): The IT team deploys a GPO "Security Baseline" that configures: MinimumPasswordLength=14, AccountLockoutThreshold=5, AuditProcessCreation=Enabled, WindowsDefender=Enabled. An attacker cannot edit GPOs unless they have Domain Admin rights, making GPO a strong enforcement mechanism.
2. Network Segmentation — VLANs / Zero Trust

Network segmentation divides the network into zones (DMZ, internal, management, guest) using VLANs and firewalls. Zero Trust architecture eliminates implicit trust — every request is authenticated and authorized regardless of source. Micro-segmentation applies firewall rules between workloads (e.g., web tier can only talk to app tier on port 8080).

Example (Segmentation): A company has VLANs: VLAN 10 (Users — 192.168.10.0/24), VLAN 20 (Servers — 192.168.20.0/24), VLAN 30 (DMZ — 10.0.0.0/24). Firewall rules: - VLAN 30 -> VLAN 20: ALLOW only HTTP/HTTPS to app servers - VLAN 10 -> VLAN 30: ALLOW only HTTPS to web servers - VLAN 10 -> VLAN 20: DENY all (users cannot directly access servers) - ALL VLANs -> WAN: ALLOW only through explicit proxy When the attacker compromises a user workstation (VLAN 10), they cannot directly reach the database server (VLAN 20) — the firewall blocks it. The attacker must pivot through the app server (VLAN 20, port 80/443 only), limiting lateral movement.
3. RBAC / ABAC

Role-Based Access Control (RBAC) assigns permissions based on job roles (e.g., "Finance_Role" can access accounting app). Attribute-Based Access Control (ABAC) uses attributes: user department, location, device compliance, time of day. ABAC is more granular and context-aware.

Example (RBAC): In AWS IAM, an RBAC policy: { "Effect": "Allow", "Action": "s3:GetObject", "Resource": "arn:aws:s3:::finance-bucket/*", "Condition": { "StringEquals": { "aws:PrincipalTag/Role": "finance" } } }. Only users tagged with "finance" role can read the finance bucket.
Example (ABAC): Access to the HR portal requires: user.department == "HR" AND device.isCompliant == true AND time.between(8:00, 18:00) AND location == "office" OR vpn.isConnected == true. An HR employee working from home without VPN at 2 AM would be denied — even with the correct role.
4. Compliance — GDPR / HIPAA / PCI-DSS

Compliance frameworks mandate security controls: GDPR (EU data protection — breach notification within 72h, data subject rights). HIPAA (US healthcare — PHI protection, access controls, audit logs). PCI-DSS (payment card data — encryption, network segmentation, quarterly scans, annual assessment).

Example (GDPR): A data breach exposes customer PII (names, emails, credit card numbers). The DPO must notify the supervisory authority within 72 hours of awareness. Notification includes: nature of breach, categories of data, estimated number of records (50,000 customers), contact details of DPO, and recommended mitigation steps. Failure to notify results in fines up to 4% of global annual turnover.
Example (PCI-DSS): Requirement 3.4: "Render PAN unreadable anywhere it is stored" — solution: tokenization or AES-256 encryption. An audit finds credit card numbers stored in plaintext in the CRM database. The QSA issues a non-compliance finding. The company must remediate within 30 days or risk losing card processing ability. Remediation: implement column-level encryption for PAN fields and truncate display (************1234).
