Elaborated Notes with Examples
1. EDR — CrowdStrike / Microsoft Defender

Endpoint Detection and Response (EDR) provides real-time monitoring, threat detection, and automated response on endpoints. CrowdStrike Falcon uses cloud-native AI with indicators of attack (IOA) detection. Microsoft Defender for Endpoint integrates with the Microsoft security stack and offers ASR rules, tamper protection, and automated investigation.

Example (CrowdStrike): Alert: "Process Injection — Remote Thread Created." Detection details: malware.exe injected into lsass.exe. CrowdStrike's IOA engine identifies OpenProcess -> VirtualAllocEx -> WriteProcessMemory -> CreateRemoteThread as a known attack pattern. Automated containment isolates the host. The analyst reviews the detection in the Falcon console, sees the MITRE mapping (T1055.001), and launches Fusion workflow to search for similar activity across all endpoints.
Example (Defender): Alert: "Suspicious PowerShell command line." ASR rule 9.3.1 blocks powershell.exe -Enc ... from winword.exe. Automated investigation runs a scan and checks for persistence changes. The analyst reviews the MDE timeline and sees the full attack chain: phishing email -> Word macro -> PowerShell -> Cobalt Strike.
2. NGFW / IDS / IPS — Snort / Suricata

Next-Generation Firewalls (NGFW) perform deep packet inspection, application identification, and TLS decryption. IDS/IPS like Snort and Suricata use signature-based and anomaly-based detection rules to identify malicious network traffic in real-time.

Example (Snort Rule):
alert tcp $HOME_NET any -> $EXTERNAL_NET 80 (msg:"ET MALWARE Cobalt Strike Beacon"; content:"|00 00 00 08 00 00 00|"; offset:4; depth:3; content:"|05 00 00 00 00 00 00 00|"; distance:0; fast_pattern; within:8; classtype:trojan-activity; sid:21001234; rev:1;)
This rule detects default Cobalt Strike beacon HTTP headers. When a match occurs, Suricata logs the alert with metadata including alert_type: trojan-activity and signature: ET MALWARE Cobalt Strike Beacon. The analyst uses the alert to trace the infected host and block the C2 IP on the NGFW.
3. WAF — Cloudflare / ModSecurity

Web Application Firewalls (WAF) protect web apps from Layer 7 attacks. Cloudflare WAF offers managed rulesets (OWASP, Cloudflare Managed) with bot management and rate limiting. ModSecurity is an open-source WAF engine using the OWASP Core Rule Set (CRS) for detecting SQLi, XSS, LFI, and RCE.

Example (ModSecurity): Log entry: ModSecurity: Access denied with code 403 (phase 2). Pattern match "(?i:union.*select.*from)" at ARGS:id. [file "/etc/modsecurity/rules/942100.conf"] [msg "SQL Injection Attack Detected"]. The analyst sees that the WAF blocked ?id=1' UNION SELECT 1,@@version,user(),4--. The rule 942100 (OWASP CRS) triggered correctly. The attack came from IP 185.130.5.201 (proxied through Tor exit node). The analyst blocks the IP and adds geo-blocking for Tor exits at the CDN level.
4. DLP & SOAR

Data Loss Prevention (DLP) monitors and prevents unauthorized data exfiltration via email, web, USB, and cloud apps. SOAR (Security Orchestration, Automation and Response) like Splunk SOAR or Palo Alto XSOAR automates incident response workflows, integrating with EDR, SIEM, ticketing, and threat intel platforms.

Example (DLP): Alert: "Sensitive Data Sent via Email." An outbound email from employee@corp.com to gmail.com contains ##CREDIT CARD## regex match for 16-digit number patterns. The DLP rule blocks the email and auto-CCs the security team. Investigation shows the employee was forwarding customer PII for "personal use." HR is notified and USB ports are disabled on the endpoint.
Example (SOAR): A phishing alert triggers a SOAR playbook: (1) Enrich sender domain via VirusTotal. (2) Check if any other employees received the same email. (3) Delete the email from all inboxes via Graph API. (4) Create a Teams notification for the SOC team. (5) Create a ServiceNow ticket with all IOCs. (6) Block the sender domain on the email gateway. Total time: 45 seconds (manual would be 20 minutes).
