Module 13: SIEM Triage for SOC
Sub-Domains / Rooms
  • Log Analysis with SIEM — Query and analyze logs across multiple sources: Windows events, firewall logs, web logs, and endpoint telemetry.
  • Alert Triage with Splunk — Use SPL to investigate alerts, build dashboards, and create correlation rules in Splunk.
  • Alert Triage with Elastic — Use Kibana and Elasticsearch for log analysis, detection rules, and alert investigation.
  • ItsyBitsy — Challenge room: investigate a web attack using SIEM data to identify the attacker and compromised data.
  • Benign — Challenge room: distinguish between benign and malicious activity using SIEM correlation.
  • Monday Monitor — Challenge room: investigate suspicious activity detected during morning monitoring.
  • Friday Overtime — Challenge room: late-night incident investigation using multiple log sources.
  • Retracted — Challenge room: analyze a retracted alert that was initially dismissed but needs re-investigation.
Example Investigation: The SIEM alerts: "Multiple failed logons followed by successful logon from foreign IP." The analyst queries Splunk:
index=windows EventID=4625 | stats count by Account_Name, Source_IP — finds 200 failed attempts on admin account from 45.33.32.156.
index=windows EventID=4624 Account_Name=admin — one successful logon from same IP.
index=windows EventID=4688 Account_Name=admin — attacker ran cmd.exe, then powershell.exe to download payload.
Conclusion: RDP brute force succeeded. Analyst isolates server, resets admin password, enables MFA.
