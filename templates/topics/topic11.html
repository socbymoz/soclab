🔹 Malware Analysis Fundamentals

Malware analysis is the study of malicious software to understand its functionality, origin, and impact. There are two main approaches:

Static Analysis (without executing):
- Hash lookup: Get SHA256, check VirusTotal → 15/70 detection as "Trojan.AgentTesla"
- Strings analysis: Extract readable text → finds URLs, IPs, registry keys, commands
- PE analysis: Check PE header, sections, imports/exports, compile time, digital signature
- File type detection: Use `file` command to verify — "invoice.pdf" might be a .exe

Dynamic Analysis (execute in sandbox):
- Process monitoring: What processes are created? (ProcMon, Process Hacker)
- Registry changes: What registry keys are modified? (RegShot — before/after comparison)
- File system changes: What files are created/modified/deleted?
- Network traffic: What connections are made? (Wireshark, INetSim)
- Memory analysis: What's injected into other processes?

🔹 Malware Detection in SOC Context

How to identify malware infections from SIEM alerts, EDR telemetry, and network traffic patterns.

Common Malware Infection Indicators in SOC:

1. Process Anomalies:
- Office apps spawning PowerShell/cmd (phishing → macro → malware)
- Unsigned binaries running from %TEMP%, %APPDATA%, or %USERPROFILE%
- Processes with no parent (run via WMI, scheduled tasks)
- Multiple instances of same process (possible bot)

2. Network Anomalies:
- Regular connections to unknown external IPs (C2 beaconing — every 60 seconds)
- DNS queries to DGA domains (random-looking domains like jd83kf92.ddns.net)
- HTTP traffic to IPs instead of domains (no host header)
- User-Agent strings that don't match the browser version
- Unusual protocols on standard ports (DNS tunnel, HTTP on non-standard ports)

3. File System Anomalies:
- Executables with hidden attributes
- Files in unusual locations (C:\Windows\SysWOW64\Tasks\svchost.exe)
- Recently modified system files
- Large numbers of files being encrypted or renamed

4. Registry Anomalies (Windows):
- New Run keys pointing to Temp or AppData
- Service entries pointing to non-standard paths
- Image File Execution Options (IFEO) debugger keys
- AppInit_DLLs with malicious DLLs

5. Behavioral Patterns:
- Beaconing (regular check-ins every 30-60 seconds)
- Reconnaissance commands (whoami, net user, ipconfig, systeminfo)
- Data staging (files collected in one folder before exfiltration)
- Defense evasion (disabling AV, deleting logs, stopping services)
YARA Rule Writing
YARA Rule Example — Detect AgentTesla:
rule AgentTesla_Detect {
meta:
description = "Detect AgentTesla InfoStealer"
author = "SOC Analyst"
date = "2026-05-21"
hash = "3f5b8c2a..."
strings:
$smtp = "SMTP" nocase
$mail_key = "mail.evil.com"
$keylog = "keylog" nocase
$screenshot = "screenshot" nocase
$persistence = "CurrentVersion\\Run"
$mz = "MZ"
condition:
$mz at 0 and 3 of ($smtp, $mail_key, $keylog, $screenshot, $persistence)
}
This rule matches any PE file (starts with MZ) that contains at least 3 of the 5 strings associated with AgentTesla.
