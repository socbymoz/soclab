Module 2: SOC Team Internals
Sub-Domains / Rooms
  • SOC L1 Alert Triage — Prioritize and classify alerts (TP vs FP vs BP), use playbooks, and follow escalation paths.
  • SOC L1 Alert Reporting — Document findings with structured reports, including timeline, IOCs, and actions taken.
  • SOC Workbooks and Lookups — Use structured workbooks to track investigations and enrich data with lookup tables.
  • SOC Metrics and Objectives — Measure SOC performance with KPIs like MTTR, alert volume, and detection rate.
  • SOC Simulator: Introduction to Phishing — Simulated phishing triage scenarios to practice real-world workflows.
Example Workflow: An L1 analyst receives a SIEM alert: "Suspicious PowerShell execution from WINWORD.EXE." The analyst checks the workbook, finds no prior incidents from this endpoint. They look up the process hash in VirusTotal — 0 detections (unknown file). They enrich with geo-IP: the C2 IP resolves to Russia. The analyst writes a report, classifies as TP (True Positive), and escalates to L2 with all evidence attached.
Key Skills Learned
  • Alert classification (TP, FP, BP)
  • Incident documentation and reporting
  • SOC metrics and performance measurement
  • Phishing triage workflows
