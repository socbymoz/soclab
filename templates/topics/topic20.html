Elaborated Notes with Examples
1. Hash Lookups

VirusTotal hash lookups return detection ratio, file metadata, community comments, and relationships. Look for: file type, compilation timestamp (is it recent? matches the claimed file?), signature info (signed by a legitimate CA?), first submission date (brand new = suspicious), and detection ratio (if 0/70 = not necessarily clean — could be custom malware).

Example: An analyst submits SHA256 a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6.
Results: 38/70 detections, first submission 2 days ago, filename invoice_2026.pdf.exe, compile time 2026-06-15 03:00:00 UTC (says it was compiled at 3 AM — unusual), file type PE32 executable (GUI) Intel 80386 (not a PDF despite the name), signing info No signing info. The detection names include Trojan.CobaltStrike, Win64/Beacon, Malicious_Behavior.SB. Community comment: "This is the JupiterLoader variant — C2 at evil-c2.ddns.net:443." The analyst blocks the hash and uploads to sandbox for behavioral analysis.
2. File Upload

VT accepts file uploads up to 650MB (registered users). Uploaded files get scanned by 70+ engines and behavioral analysis sandbox (VirusTotal Jujubox). Results include: process tree, file operations, registry changes, network connections, and MITRE ATT&CK tactics mapping.

Example: The analyst uploads suspicious_sample.exe. After analysis:
Detection tab: 45/70 engines detect as malicious. Tags: peexe, packed-upx, overlay.
Details tab: SHA256, MD5, SHA1, file type, size, first/last submission, compilation timestamp, TRiD file type identifier: Win32 Executable Generic.
Behaviour tab: Sandbox report shows:
- CreateProcess: C:\Windows\System32\cmd.exe /c whoami
- CreateProcess: C:\Windows\System32\powershell.exe -enc ...
- Network: HTTP GET evil-c2.ddns.net/beacon — 200 OK
- File: C:\Users\admin\AppData\Local\svchost.exe written
- Registry: HKCU\...\Run\WindowsUpdate created
MITRE mapping: T1059.001 (PowerShell), T1071.001 (Web Protocols), T1547.001 (Registry Run Keys).
3. Graphs / Relationships

The Relations tab shows connected entities: contacting IPs, domains, dropped files, parent/child files. This graph view reveals infrastructure relationships and helps identify related samples. The "Graph" visualization is interactive with clickable nodes.

Example: Viewing the Relations tab for the malware hash reveals:
- Contacted IPs: 185.130.5.201 (18 detections), 203.0.113.5 (6 detections)
- Contacted domains: evil-c2.ddns.net (15 detections), cdn-verify.net (8 detections)
- Dropped files: svchost.exe (SHA256: b2c3d4e5... — 32/70 detections)
- Parent relationships: invoice.docm (SHA256: c3d4e5f6... — 25/70 detections, VBS/DownLoader)
- Graph view: invoice.docm -> malicious.exe -> svchost.exe + evil-c2.ddns.net + 185.130.5.201
The analyst clicks on evil-c2.ddns.net and sees it resolves to 5 different IPs across 3 months — indicating a dynamic C2 infrastructure probably using Fast Flux DNS. This helps the analyst build a complete picture of the attacker's infrastructure.
4. API Automation

VirusTotal provides a REST API for automated queries: hash lookups, file uploads, URL scans, IP/domain reports. Python library vt-py simplifies integration. API key scopes control access. Rate limits apply (4 requests/min for free, higher for premium).

Example Python Script:
import vt
client = vt.Client("YOUR_API_KEY")
hash = "a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6"
file = client.get_object(f"/files/{hash}")
print(f"Detection ratio: {file.last_analysis_stats['malicious']}/70")
print(f"Type: {file.type_description}")
print(f"Tags: {', '.join(file.tags)}")
# Get relationships
relations = client.get_object(f"/files/{hash}/relationships/contacted_domains")
for domain in relations.data:
  print(f"Domain: {domain.id}")
client.close()
This script can be integrated into a SOAR playbook. When a new phishing alert fires, the SOAR automatically queries VT for the attachment hash, enriches the alert with detection ratio and tags, and auto-closes if 0/70 (low confidence) or escalates if 10+/70 detections.
