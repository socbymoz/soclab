🔹 Windows Logging for SOC

Windows Event Log is the primary source of forensic data on Windows. There are three main log channels plus Sysmon for advanced monitoring.

Critical Windows Event IDs Every SOC Analyst Must Know:

| Event ID | Description | Why It Matters | |----------|-------------|----------------| | 4624 | Successful Logon | Correlate with failed attempts before it = brute force success | | 4625 | Failed Logon | Brute force attacks, password guessing | | 4634 | Logoff | Track session duration | | 4648 | Logon with Explicit Credentials | Runas or scheduled task — possible lateral movement | | 4672 | Admin Logon | Someone got admin rights | | 4688 | Process Creation | See every process launched | | 4698 | Scheduled Task Created | Persistence mechanism | | 4700 | Scheduled Task Enabled | Attacker activates backdoor | | 4720 | User Account Created | Possible backdoor account | | 4732 | User Added to Group | Privilege escalation (added to Admin) | | 4776 | Credential Validation | Domain authentication — track NTLM usage | | 5140 | SMB Share Accessed | Lateral movement via SMB | | 5156 | Windows Filtering Platform | Connection allowed — network whitelist enforcement | | 7045 | Service Installed | Backdoor installed as a service |

🔹 Windows Threat Detection 1: Brute Force & Account Misuse

Brute Force Detection:
index=windows EventID=4625 | stats count by Account_Name, Source_IP | where count > 20 | sort -count
→ Find accounts with 20+ failed logins. If followed by EventID 4624 (success), the attacker cracked the password.

Example — RDP Brute Force:
EventID 4625 × 500 on account "admin" from IP 45.33.32.156 over 5 minutes
Then EventID 4624 — successful login on same account from same IP
Then EventID 4688 — cmd.exe → powershell.exe → turned off Windows Defender
Then EventID 4698 — scheduled task created to run malware every hour
Then EventID 3 (Sysmon) — network connection to 45.33.32.156:4444

Timeline of the attack:
14:32:00 — Start of brute force (500 attempts in 5 min)
14:37:15 — Successful login (password was "Admin@123")
14:37:30 — Disabled Windows Defender via PowerShell
14:38:00 — Downloaded Cobalt Strike beacon
14:38:30 — Created persistence (scheduled task)
14:39:00 — C2 beacon established

🔹 Windows Threat Detection 2: Process Injection & Persistence

Common Process Injection Detection:

1. CreateRemoteThread (Sysmon Event 8):
RuleName: technique_id=T1055.001, technique_name=Process Injection
UtcTime: 2026-05-21 14:45:12.345
SourceProcessGUID: {A1B2C3...}
SourceImage: C:\Users\victim\AppData\Local\Temp\malware.exe
TargetImage: C:\Windows\System32\svchost.exe
NewThreadId: 4567
→ Malware injected code into svchost.exe to hide its activity

2. DLL Hijacking Detection:
Check Sysmon Event 7 (Image Loaded) for DLLs loaded from unusual paths:
ImageLoaded: C:\Users\victim\AppData\Local\Temp\version.dll
→ Normally version.dll loads from C:\Windows\System32 — loading from Temp is abnormal

3. Persistence Detection (Sysmon Event 12/13):
Registry Run Key modification:
EventType: SetValue
TargetObject: HKLM\Software\Microsoft\Windows\CurrentVersion\Run\MalwareService
Details: C:\Users\victim\AppData\Local\Temp\malware.exe
→ Common persistence — malware adds itself to Run key to start on boot

🔹 Windows Threat Detection 3: Ransomware & Lateral Movement

Ransomware Detection Indicators:
1. File creation events — massive number of .encrypted, .locked, .crypted files
2. Ransom note dropped — "README_TO_DECRYPT.txt" or similar in every directory
3. Volume Shadow Copy deletion — vssadmin.exe delete shadows /all /quiet
4. Boot configuration modification — bcdedit.exe /set {default} recoveryenabled No
5. Process tree: explorer.exe → cmd.exe → vssadmin.exe

Lateral Movement Detection (Pass-the-Hash):
EventID 4624 (Logon Type 3 — Network) from IP 10.0.1.50 to 10.0.1.100
Logon Process: NtLmSsp
Authentication Package: NTLM
When combined with Event 4776 and Event 4672 (admin logon), this indicates potential pass-the-hash attack — attacker captured a hash and reused it to authenticate to another server.

Detection Query (Splunk):
index=windows (EventID=4624 LogonType=3) OR (EventID=4776)
| stats count by Account_Name, Source_IP, dest_ip
| where count > 1 AND count < 10
→ Shows accounts that authenticated from one host to multiple destinations — lateral movement pattern
Windows Security Checklist
Daily Monitoring Queries for Windows SOC:
1. Brute force: EventID=4625 count > 10 in 5 min → alert
2. Admin activity: EventID=4672 during off-hours (22:00-06:00) → investigate
3. Suspicious processes: EventID=4688 where ParentImage contains WINWORD, EXCEL, OUTLOOK and Image contains powershell, cmd, wscript, cscript
4. New accounts: EventID=4720 → review for unauthorized account creation
5. New services: EventID=7045 → review for unauthorized service installation
6. Group membership changes: EventID=4732 where TargetUserName=Administrators → privilege escalation
