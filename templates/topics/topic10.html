Module 10: Linux Security Monitoring
Sub-Domains / Rooms
  • Linux Logging for SOC — Understand syslog, auth.log, secure log, dmesg, and auditd for monitoring Linux systems.
  • Linux Threat Detection 1 — Detect SSH brute force, user enumeration, and unauthorized sudo usage from auth logs.
  • Linux Threat Detection 2 — Identify reverse shells, cron job abuse, and file integrity changes on Linux endpoints.
  • Linux Threat Detection 3 — Investigate kernel-level threats, rootkits, and persistence via systemd services and LD_PRELOAD.
Example Detection:
/var/log/auth.log shows 3000 failed SSH attempts from IP 45.33.32.156 in 5 minutes — brute force attack.
/var/log/syslog shows a cron job running every minute: */1 * * * * /tmp/.hidden/updater.sh — persistence.
audit.log shows auid=1000, uid=0, comm="bash" — privilege escalation to root.
The analyst checks .bash_history and finds the attacker downloaded a reverse shell, escalated to root via CVE-2021-4034 (PwnKit), and installed a systemd service for persistence.
Key Skills Learned

  • Reverse shell and persistence detection
