Elaborated Notes with Examples
1. CIA Triad

The CIA Triad is the foundation of all cybersecurity.

Confidentiality means only authorized people can access data — enforced via encryption (AES-256), access controls (RBAC), and the principle of least privilege.

Example: A hospital encrypts patient records with AES-256 so only the assigned doctor can read them. The doctor's badge number maps to an RBAC role that grants read access only to their patients' records.

Integrity ensures data hasn't been tampered with — verified through hashing (SHA-256), digital signatures, and audit logs.

Example: A bank transaction log uses SHA-256 hashes chained together (blockchain-like). If an attacker modifies a transaction amount from $100 to $10,000, the hash chain breaks and the tampering is immediately detected during the next audit.

Availability means systems are up when needed — achieved via redundancy (RAID, failover clusters), DDoS protection, and regular backups.

Example: An e-commerce site deploys servers across 3 AWS availability zones with an Elastic Load Balancer. When one zone goes down during a DDoS attack, traffic automatically shifts to the remaining two zones. The site stays online.
2. Types of Threats & Threat Actors

APT (Advanced Persistent Threat): Nation-state actors who stay hidden for months.

Example: APT29 (Cozy Bear) breached SolarWinds in 2019 and remained undetected for 9 months. They compromised the software build pipeline to inject backdoors into Orion updates, affecting 18,000 customers.

Ransomware: Encrypts data for ransom.

Example: LockBit ransomware group breached a manufacturing firm via RDP brute force, moved laterally to 200 servers, encrypted 400+GB of CAD files, and demanded $5M in Bitcoin. The company paid $2.5M after negotiation.

Insider Threat: An employee or contractor misusing access.

Example: A sysadmin at a telecom company copied 50K customer records (names, SSNs, call logs) to a USB drive over 3 months before leaving for a competitor. The activity was only discovered during an audit triggered by a competitor launching suspiciously targeted marketing campaigns.

Phishing: Social engineering via email.

Example: An attacker posed as the CEO and emailed finance@company.com: "I'm in a sensitive board meeting. Wire $200K to the attached vendor account immediately. I'll sign the paperwork when I'm back." The email came from a spoofed domain (c0mpany.com with a zero). Finance wired $200K. The real CEO never sent that email.
3. SOC Tiers & Responsibilities

L1 Analyst (Triage): Monitors alerts, validates if they're real threats (TP vs FP), escalates confirmed incidents. They use SIEM dashboards, follow runbooks, and handle ~80% of alerts.

Skills: Log reading, basic triage, runbook following.
Example: An L1 sees an alert: "Suspicious PowerShell from unknown parent." They check the SIEM, see the process is running from %TEMP% (unusual), and also find that the parent process is Microsoft Word (very unusual). They escalate to L2 as a confirmed True Positive.

L2 Analyst (Investigation): Deep-dives into escalated incidents, performs forensics, contains threats.

Skills: Forensics, malware analysis, network analysis.
Example: L2 takes the escalated case. They query the EDR for related IOCs, find the PowerShell contacted 45.33.32.156:4444 (unknown external IP). They isolate the endpoint, capture a memory dump, and extract the PowerShell script. The script downloads and executes a Cobalt Strike beacon.

L3 Analyst (Expert/Threat Hunter): Hunts for advanced threats, reverse-engineers malware, builds detection rules.

Skills: Reverse engineering, exploit analysis, automation.
Example: L3 reverse-engineers the Cobalt Strike beacon in IDA Pro, identifies the C2 protocol (HTTPS with custom user-agent), extracts the embedded config, and writes a YARA rule + Suricata IDS signature. They also search the environment for similar beacons using the new YARA rule.
4. Incident Response Lifecycle (NIST)

1. Preparation: Build playbooks, deploy tools (EDR, SIEM), train staff.

A company runs tabletop exercises every quarter. Each team member has a printed IR playbook with clear steps for ransomware, phishing, data breach. EDR is deployed to all 5,000 endpoints. SIEM ingests firewall, EDR, and Windows event logs.

2. Detection & Analysis: Identify suspicious activity through alerts, logs, and reports.

A spike in failed logins (Event ID 4625: 50 attempts in 60 seconds from IP 45.33.32.156) triggers a brute force alert. The SIEM correlation rule tracks: count(EventID=4625) > 10 within 5 minutes from same source IP.

3. Containment, Eradication & Recovery: Stop the attack, remove malware, restore from clean backup.

Contain: Isolate infected VM via EDR network quarantine (blocks all traffic except to SIEM).
Eradicate: Kill malicious processes, delete persistence registry keys (HKLM\...\Run\MaliciousService), remove scheduled tasks ("GoogleUpdateTask").
Recover: Reimage from known-good backup taken 24 hours before the incident. Verify by running full EDR scan — 0 detections.

4. Post-Incident Activity: Write report, identify gaps, implement improvements.

Write incident report: "Root cause — weak RDP password. 5 servers exposed to internet. No MFA. No failed login alerting."
Improvements: Deploy MFA on all external RDP. Close RDP ports at firewall. Create rate-limiting rule for failed logins. Deploy privileged access workstation (PAW) for admins.
5. Security Frameworks

NIST CSF: Identify → Protect → Detect → Respond → Recover. Used by US government and critical infrastructure.

A power utility implements NIST CSF: Identifies all OT/SCADA assets, protects them with network segmentation, detects anomalies via ICS-specific IDS, responds with OT-specific playbooks, and recovers from air-gapped backups.

ISO 27001: International ISMS standard requiring risk assessments, security controls, continuous improvement.

A fintech company achieves ISO 27001 certification by documenting all security policies, conducting annual risk assessments, implementing 114 controls from Annex A, and passing an external audit.

MITRE ATT&CK: Knowledge base of adversary tactics and techniques for threat modeling and detection.

A SOC uses ATT&CK to map detection coverage. They discover they have no detections for T1059.003 (PowerShell). They deploy PowerShell module logging (Event ID 4103, 4104) and create alerts for suspicious command-line patterns.
