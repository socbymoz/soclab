Module 5: Phishing Analysis
Sub-Domains / Rooms
  • Phishing Analysis — Analyze email headers (SPF, DKIM, DMARC), inspect URLs safely, extract and analyze attachments.
  • Phishing Prevention — Deploy email filtering rules, user awareness training, DMARC enforcement, and reporting mechanisms.
  • Real-Phish Investigations — Investigate real-world phishing campaigns, trace attacker infrastructure, and identify compromised accounts.
Example Analysis: An analyst receives a reported phishing email claiming to be from "security@paypa1.com". Header analysis shows: SPF=fail (sending IP 185.130.5.201 not authorized), DKIM=fail (signature mismatch), DMARC=fail (p=reject). The URL hxxps://paypa1.com/login resolves to 45.33.32.156 hosted in Russia. The attachment "Invoice.docm" contains a malicious VBA macro that downloads a PowerShell payload. The analyst blocks the domain, IP, and file hash, and sends a company-wide warning.
Key Skills Learned
  • Email header analysis (SPF, DKIM, DMARC)
  • Safe URL and attachment analysis
  • Phishing kit identification
  • Incident response for phishing campaigns
