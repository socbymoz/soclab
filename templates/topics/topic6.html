Elaborated Notes with Examples
1. Server-Side Request Forgery (SSRF)

SSRF exploits a web application to make requests to internal resources. Attackers target http://169.254.169.254/ (cloud metadata), http://localhost:9200/ (Elasticsearch), or internal services via URL parameters like ?url=, ?file=, ?redirect=.

Example: A web app's profile image feature: GET /fetch?url=http://evil.com/photo.jpg. The attacker changes to http://169.254.169.254/latest/meta-data/iam/security-credentials/admin and receives AWS access keys in the response. Logs show the source IP is the internal load balancer (10.0.0.1), proving the request was made server-side.
2. Insecure Deserialization

Insecure deserialization occurs when untrusted data is deserialized without validation, leading to RCE. Common in Java (ObjectInputStream, readObject), PHP (unserialize), Python (pickle), and .NET (BinaryFormatter). Detection involves monitoring for serialized stream magic bytes and anomalous child processes.

Example: A Java application sends cookie values like Cookie: user=ACED0005737200116A6176612E7574696C2E486173684D6170. The hex ACED0005 is Java serialization magic bytes. A ysoserial payload with CommonsCollections4 chain triggers Runtime.exec("calc.exe"). WAF logs show the cookie size jumped from 120 bytes to 4500 bytes — a clear anomaly.
3. API Abuse

Common API attacks include broken authentication (missing JWT validation), lack of rate limiting (account takeover via credential stuffing), and IDOR (Insecure Direct Object Reference) where user A modifies /api/user/12345 to access user B's data.

Example (IDOR): API log: GET /api/v2/orders/ORD-00199 from user alice@corp.com is fine. But the same user then sends GET /api/v2/orders/ORD-00200, ORD-00201, through ORD-00500 in rapid succession (200 req/min). The API returns order details including PII (names, credit card last-4) for other customers. The analyst sees the API lacks authorization checks — only the JWT user ID is verified, not resource ownership.
4. WAF Bypass Techniques

Attackers bypass WAF rules using encoding (URL double-encoding, Unicode normalization), HTTP parameter pollution (HPP), case variations, comment injection (/**/), null bytes, and content-type switching (JSON vs form-data).

Example (SQLi bypass): The attacker sends ?id=1'/*!14400UNION*/+/*!14400SELECT*/+1,2,3--. MySQL treats /*!14400 code */ as conditional SQL — the version number (14400 > MySQL version) causes the server to execute the code, while the WAF regex /UNION\s+SELECT/i fails to match because of the comment. The request gets through and the attacker extracts data.
