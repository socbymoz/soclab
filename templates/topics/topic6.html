Module 6: Network Traffic Analysis
Sub-Domains / Rooms
  • Wireshark Basics & Deep Dives — Capture and analyze network packets, apply filters, follow TCP streams, and detect anomalies.
  • TShark Challenge I: Teamwork — Command-line packet analysis with TShark to extract specific data from PCAPs.
  • TShark Challenge II: Directory — Advanced TShark challenges involving directory traversal detection and data extraction.
Example: An analyst opens a PCAP in Wireshark. They apply a filter `http.request` and see a GET request to `/login.php?id=1' OR '1'='1` — SQL injection attempt. Following the TCP stream reveals the attacker dumped the database. They also see periodic HTTPS connections to 45.33.32.156:443 every 60 seconds — C2 beaconing. They use TShark to extract all connections to this IP: `tshark -r capture.pcap -Y "ip.addr==45.33.32.156"`.
Key Skills Learned
  • Wireshark filters and packet analysis
  • TShark command-line packet processing
  • Detecting C2 beaconing and data exfiltration
  • Protocol analysis (HTTP, DNS, TLS)
