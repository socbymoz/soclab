Module 6: Network Traffic Analysis

Network traffic analysis is a core SOC skill. You'll learn to capture, filter, and analyze packets using Wireshark and TShark to detect malicious activity.

Sub-Domains / Rooms

🔹 Wireshark Basics & Deep Dives

Wireshark is the industry-standard network protocol analyzer. Key skills every SOC analyst needs:

Essential Wireshark Filters:
ip.addr == 45.33.32.156 — Show all traffic to/from a specific IP
http.request — Show only HTTP request packets
tcp.port == 443 — Show only HTTPS traffic
dns.qry.name contains "evil" — Find DNS queries containing "evil"
http.request.uri contains "cmd=" — Find web requests with command execution
tls.handshake.type == 1 — Show TLS Client Hello (identify JA3 fingerprints)
data.len > 1000 — Show packets with large data payloads (possible exfiltration)
!(arp or dns or dhcpv6) — Hide noise protocols

Detecting C2 Beaconing in Wireshark:
1. Filter: ip.addr == 10.0.1.50 && tcp.flags.syn == 1 && tcp.flags.ack == 0
2. Look for SYN packets to the same destination at regular intervals (e.g., every 60 seconds)
3. Check: Go to Statistics → Flow Graph → TCP to visualize connection timing
4. C2 beacon pattern: Regular, periodic connections to external IP, small data exchange, often on unusual ports (4444, 8080, 8443)

🔹 TShark Challenge I: Teamwork

TShark is the command-line version of Wireshark. Essential for automated analysis, server environments, and large PCAP processing.

TShark Commands for SOC:
tshark -r capture.pcap -Y "http.request" -T fields -e http.host -e http.request.uri
→ Extract all HTTP requests — see every URL visited

tshark -r capture.pcap -Y "dns" -T fields -e dns.qry.name | sort | uniq -c | sort -rn
→ List all DNS queries sorted by frequency — helps identify DGA domains

tshark -r capture.pcap -Y "ip.addr==45.33.32.156" -T fields -e frame.time -e ip.src -e ip.dst -e tcp.srcport -e tcp.dstport
→ Extract all connections to a specific IP with timestamps

tshark -r capture.pcap -z conv,tcp | sort -k5 -rn | head -10
→ Show top 10 TCP conversations by bytes transferred — detect data exfiltration

Challenge Scenario — Teamwork:
Your team receives a PCAP of a suspected breach. Use TShark to:
1. Identify the attacker's IP address (most connections, scanning behavior)
2. Find what vulnerability was exploited (check HTTP requests for SQLi, path traversal, etc.)
3. Extract the payload downloaded from the C2 server
4. Determine what data was exfiltrated (look at large outbound transfers)
5. Build a timeline of attacker actions using frame timestamps

🔹 TShark Challenge II: Directory

Advanced TShark challenges focused on directory traversal attacks and complex data extraction from PCAPs.

Directory Traversal Detection:
tshark -r capture.pcap -Y "http.request.uri contains \"../\" or http.request.uri contains \"..%2f\" or http.request.uri contains \"%2e%2e/\""
→ Find directory traversal attempts in HTTP requests

Extracting Transferred Files:
tshark -r capture.pcap --export-objects "http,./extracted_files"
→ Extract all files transferred via HTTP from the PCAP

tshark -r capture.pcap -Y "data.data" -T fields -e data.data | xxd -r -p > extracted_file.bin
→ Extract raw binary data from packets

Challenge Scenario — Directory:
You're investigating a web server compromise. The PCAP shows:
1. Initial reconnaissance: GET /../../etc/passwd HTTP/1.1 — attacker found path traversal
2. File read: GET /../../../etc/shadow — attacker read password hashes
3. File upload: POST /uploads/cmd.aspx PHP content-type — web shell uploaded
4. Command execution: GET /uploads/cmd.aspx?cmd=whoami — attacker ran commands
5. Data exfiltration: GET /uploads/cmd.aspx?cmd=type%20C:\Users\Administrator\Desktop\passwords.txt

Your tasks: Extract all files, reconstruct the attack timeline, identify stolen data, and determine the vulnerability's root cause.
Network Traffic Analysis Methodology
SOC Network Investigation Checklist:
1. Identify suspicious IP: Check firewall logs, proxy logs, or SIEM alerts for unusual outbound connections
2. Extract PCAP: If available, capture traffic at the endpoint or network TAP
3. Find the beacon: Look for regular periodic connections to external IPs (60s, 120s intervals)
4. Analyze protocol: Is this HTTP, HTTPS, DNS, ICMP? Unusual protocol on unusual port = suspicious
5. Check DNS: DGA domains look random (e.g., jd83kf92.ddns.net), often use .tk, .ml, .ga TLDs
6. Look for exfiltration: Large outbound transfers, especially during off-hours (2-4 AM)
7. Correlate with endpoint: Does the network traffic match process creation events on the host?
