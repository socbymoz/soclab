Elaborated Notes with Examples
1. Correlating Web, Firewall, and WAF Logs

Correlation is essential — the web server access log shows the HTTP request, the WAF log shows which rules matched, and the firewall log shows the source/destination IPs and ports. Align timestamps (±1s) across sources to build a complete picture of an attack flow.

Example: Web log: 192.168.1.5 - - [17/Jun/2026:10:15:30] "GET /shell.php?cmd=id" 200 45. WAF log: 192.168.1.5 - RULE 933210 (PHP Injection) - Blocked - Action: LOG (configured as monitor-only). Firewall log: ALLOW TCP 192.168.1.5:4444 -> 10.0.0.10:443 (C2 beacon outbound). By correlating, the analyst confirms the WAF misconfiguration (detect-only mode) and the webshell access.
2. Finding the Initial Attack Vector

Determining how the attacker first gained access is critical for remediation. Look at earliest anomalous event — the first alert triggered, the first successful exploit attempt, or the first connection to an external IP at the time of the incident. Check web logs, VPN logs, and email gateway logs.

Example: The incident timeline shows the first alert at 09:00, but web logs reveal a SQLi attempt at 08:45: GET /product?id=1'+UNION+SELECT+1,@@version,user(),4-- with response 200. At 08:47 the webshell upload: POST /admin/upload.php (cmdshell.asp). At 08:50 the first C2 beacon to evil-c2.ddns.net:8443. The SQLi payload matched WAF rule 942100 in log-only mode — the initial vector was a SQL injection that allowed the attacker to upload a webshell.
3. Post-Compromise Actions

After initial access, the attacker typically performs discovery (whoami, net users), lateral movement (PsExec, WMI, RDP), privilege escalation, persistence, and data exfiltration. Each step leaves log artifacts in the SIEM, Sysmon, and AD logs.

Example (timeline): 09:00 — Event ID 1 (Sysmon): cmd.exe /c whoami
09:02 — Event ID 3 (Sysmon): net use \\DC01\IPC$ /user:admin
09:05 — Event ID 4688: wmic /node:DC01 process call create "cmd.exe"
09:10 — Event ID 4624: LogonType 3 (admin account from WEB01 to DC01)
09:15 — Event ID 4663: Access to \\DC01\C$\Users\*\Desktop\passwords.txt
09:30 — Firewall: 500MB outbound from DC01 to 203.0.113.4:443 (rclone)
4. Data Exposure Scope & Root Cause Analysis

Scope assessment determines the blast radius: which systems, accounts, and data were accessed. Root cause analysis identifies the underlying vulnerability (missing patch, misconfiguration, weak password) to prevent recurrence. Document findings for the PIR.

Example (Scope): 3 servers compromised (WEB01, DB01, DC01), 12 user accounts stolen including 2 domain admins, 5 customer databases (50K PII records) accessed, 200GB data exfiltrated. Root cause: Apache Struts CVE-2023-34362 on WEB01 (unpatched since April 2023). WAF rule for Struts OGNL injection was in monitor-only mode. Recommendation: patch all Struts instances, enable WAF in blocking mode, deploy EDR on all servers.
